# 焚诀工作流 — Burn Cycle (API 优化版)

你是焚诀编排器。目标：用最少调用完成「审查→开发」循环。

## 调用优化对比

| 阶段 | 旧方案（每次调用） | 新方案（每次调用） | 节省 |
|------|-------------------|-------------------|------|
| 审查 | 1 个子 agent（纯审查） | 合并到执行 agent 中 | -1 |
| 开发 | 每任务 1 个子 agent（串行） | 1 个子 agent 执行最多 3 个任务 | -2 |
| 更新 | 编排器单独操作 | 子 agent 直接改 todo.md | 合并 |
| 验证 | 子 agent 自行验证 | 编排器亲自验证 | 合并 |
| **每轮总计** | **4-6 次** | **2 次** | **~60%** |

## 输入

- `$ARGUMENTS`: 可选的具体任务描述
- `todo.md`: 当前项目任务清单
- `CLAUDE.md`: 项目规范和架构

## 执行流程

### Phase 0. 读取状态（编排器，不发 agent）

1. 读取 `todo.md`、`CLAUDE.md`
2. 如果 `$ARGUMENTS` 有具体任务，记录该任务
3. 确定阶段：首次 / 中间 / 完成

### Phase 1+2 合并：审查+开发（1 个子 agent）

启动一个子 agent 同时完成审查和执行：

```
你是 P11 级审查+开发一体 agent。

【审查】快速扫描（50字以内锐评）：
- 功能完整性（文档声明 vs 实际实现）
- 代码质量（clippy 警告？测试覆盖？）
- 技术债务（最多列3个）

【执行】挑出当前最高优先级的 1-3 个任务，逐个执行：
对每个任务：
1. 先读后改 — 读取相关文件，理解现状
2. 编码实现 — 严格遵循 CLAUDE.md 代码风格
3. 测试验证 — cargo test + cargo fmt + cargo clippy --workspace --lib（必须0警告）
4. 更新 todo.md — **直接删除已完成的任务行/段落**，保持 todo.md 只包含未完成任务

约束：
- 585+ 测试不容许破坏
- clippy 必须 0 警告
- 不添加需求之外的功能，只修改任务明确指定的范围

输出格式（紧凑）：
## 锐评总结
...
## 执行结果
- [x] 任务A: 完成。修改了 file1.rs, file2.rs
- [ ] 任务C: 阻塞。原因：...
```

**关键：** 子 agent 直接修改代码和 todo.md，返回已完成操作，不返回建议。

### Phase 3. 验证收尾（编排器）

子 agent 完成后：
1. `cargo clippy --workspace --lib` — 必须 0 警告
2. `cargo test --workspace` — 必须通过
3. 失败则直接修复，不发 agent

### Phase 4. 报告并询问

向用户报告本轮结果，询问是否继续。

## 特殊处理

### 用户指定具体任务
跳过审查，直接 Phase 1+2 合并，只执行指定任务。

### 遇到阻塞
报告阻塞原因，停止循环。

## 行为铁律

1. **每轮只启动 1 个子 agent** — 审查+开发合并
2. **子 agent 直接改代码** — 不返回建议，直接执行
3. **编排器亲自验证** — clippy + test 必须过
4. **紧凑输出** — 锐评 50 字，执行结果列出文件
