### 交付物

| 文件 | 说明 |
|------|------|

### 仪表板面板

1. **阶段吞吐量 (5 分钟平均)** - 实时监控各阶段处理速率
2. **阶段耗时分布 (P50/P95/P99)** - 分析性能瓶颈
3. **阶段超时率 (5 分钟平均)** - 监控不稳定阶段
4. **阶段取消率 (5 分钟平均)** - 监控快速失败频率
5. **阶段总完成数** - 统计卡片
6. **阶段总超时数** - 统计卡片
7. **阶段总取消数** - 统计卡片
8. **阶段平均耗时 (5 分钟)** - 统计卡片

### Prometheus 告警规则

| 告警名称 | 触发条件 | 严重级别 |
|----------|----------|----------|
| HighStageTimeoutRate | 超时率 > 10% 持续 2 分钟 | Warning |
| HighStageCancelRate | 取消率 > 20% 持续 2 分钟 | Warning |
| HighStageLatencyP99 | P99 耗时 > 2s 持续 5 分钟 | Warning |
| StageStopped | 5 分钟内无完成请求 | Critical |
| ZeroCopyPerformanceDegradation | 零拷贝性能退化 > 100μs | Warning |

### 快速启动

```bash
# Windows
monitoring\start-monitoring.bat

# Linux/Mac
monitoring/start-monitoring.sh

# 访问地址
# - Grafana: http://localhost:3000 (admin/admin123)
# - Prometheus: http://localhost:9090

### 测试用例

| 测试名称 | 说明 | 阈值 |
|----------|------|------|
| `test_zero_copy_performance` | 1000 次 Arc::clone() 应在 100μs 内 | < 100μs |
| `test_zero_copy_data_size_independence` | Arc::clone() 与数据量无关 | 差异 < 50% |
| `test_deep_copy_performance_baseline` | Vec::clone() 基线测试 | < 100ms |
| `test_zero_copy_vs_deep_copy_advantage` | 零拷贝优势验证 | > 100 倍 |
| `test_zero_copy_concurrent_performance` | 并发零拷贝性能 | < 5ms |
| `test_arc_memory_overhead` | Arc 内存开销验证 | ≤ 16 字节 |

### 测试结果

```

### 性能数据

- **零拷贝性能**: 1000 次 Arc::clone() = 12-13μs
- **深拷贝性能**: 100 次 Vec::clone() 100KB = 1.5-4ms
- **性能优势**: **2000 倍+**（零拷贝 vs 深拷贝）
- **数据量无关性**: 100KB/1MB/10MB 都是 ~12μs

## ✅ 建议 3: 添加 CI/CD 集成

**优先级**: P2  
**工时**: 4 小时  
**状态**: ✅ 完成

### 交付物

| 文件 | 说明 |
|------|------|
| `.github/workflows/performance.yml` | 性能回归检测 GitHub Action |
| `crates/orchestrator/examples/metrics_server.rs` | Prometheus 指标暴露示例 |

### CI/CD 工作流

#### performance.yml - 性能回归检测

**触发条件**:
- Push 到 main/develop 分支
- Pull Request 到 main 分支

**检测项目**:
1. **零拷贝性能回归** (`zero-copy-perf`)
   - 运行零拷贝基准测试
   - 运行性能回归测试
   
2. **取消信号传播延迟测试** (`cancel-signal-perf`)
   - 100 接收者传播延迟 < 1ms
   - 1000 接收者传播延迟 < 10ms

3. **关键场景性能验证** (`critical-benchmarks`)
   - Topology 小/中/大规模场景

4. **性能报告生成** (`performance-report`)
   - 汇总所有测试结果
   - 生成 GitHub Step Summary

### 指标暴露示例

运行指标服务器：

```bash
cargo run -p orchestrator --example metrics_server
```

访问端点：
- **指标**: http://localhost:8080/metrics
- **健康检查**: http://localhost:8080/health
- **就绪检查**: http://localhost:8080/ready
