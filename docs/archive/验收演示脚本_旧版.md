## 常见问题解答（Q&A 准备）

### 核心功能问题

**Q1: 支持哪些 PDF 格式？**

> "系统支持以下 PDF 格式：
> - 矢量 PDF：直接解析，lopdf 提取路径
> - 扫描版 PDF：内置矢量化引擎自动转换（**仅适用于线条清晰的图纸**）
> - 图片型 PDF：内置矢量化引擎自动转换（**仅适用于线条清晰的图纸**）
>
> **重要限制说明**：当前矢量化功能适用于线条清晰的图纸。对于复杂扫描图纸（存在阴影/折痕/褪色、虚线/中心线/剖面线混合、文字标注与线框重叠等情况），建议先转换为 DXF 格式。完整图像预处理和线型识别能力计划于 P2 阶段（验收后 4-6 周）增强。"

### 🟡 一般问题应对

**Q3: 支持哪些 DXF 版本？**

> A: 支持 AC1015 (AutoCAD 2000) 及以上版本，覆盖绝大多数建筑图纸。

**Q4: 最大支持多大的文件？**

> A: 基准测试显示 100K 线段可以稳定处理，实际限制取决于系统内存。
> 推荐处理 < 10,000 线段的建筑图纸（典型报告厅/会议室规模）。

**Q5: 如何集成到现有系统？**

> A: 提供三种集成方式：
> 1. Rust API 直接调用（推荐）
> 2. 命令行工具（CLI）
> 3. HTTP 服务（P2 阶段提供）

**Q6: 性能怎么样？**

> A: 基准测试数据：
> - 100 线段：< 1ms
> - 1,000 线段：< 10ms
> - 10,000 线段：< 100ms
> - 100,000 线段：~1s

**Q7: 错误恢复建议怎么用？**

> A: 当验证失败时，系统会自动生成恢复建议。例如：
> ```
> 错误：检测到环未闭合，缺口距离为 0.80mm
> 建议：
>   1. [优先级 10] 增大吸附容差到 1.20mm
>   2. 使用 InteractSvc 桥接缺口
>      （建议配置：topology.snap_tolerance_mm = 1.2）
> ```
