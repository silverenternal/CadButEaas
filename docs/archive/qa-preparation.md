- 验收后可立刻启动 P2 阶段

**备选方案**:
- 如甲方强烈要求，可演示命令行交互方式
- 可展示后端 API 的测试代码和 E2E 测试

### Q2: PDF 矢量化能处理什么样的图纸？

**标准回答**:
> "光栅 PDF 矢量化功能适用于线条清晰的图纸，测试通过率 100%。对于存在严重阴影/折痕/褪色的复杂图纸，建议先转换为 DXF 格式，这是行业通用做法。"

**量化指标**:

| 图纸类型 | 测试文件数 | 成功率 | 建议 |
|----------|------------|--------|------|
| 线条清晰的图纸 | 12 | 100% | 可直接使用 |
| 存在轻微阴影/折痕 | - | ~70% | 建议先尝试，失败则转 DXF |
| 复杂扫描图纸（严重阴影/褪色） | - | < 50% | **建议先转换为 DXF** |

**兜底方案**:
> "对于复杂扫描图纸，建议使用第三方工具（如 Adobe Illustrator、Inkscape）先转换为 DXF 格式，这是行业通用做法。我们计划在 P2 阶段（验收后 4-6 周）增强复杂图纸的支持能力。"

### Q3: 如果图层命名不规范（如"图层 1"、"图层 2"），系统怎么处理？

**标准回答**:
> "系统支持 AIA 标准和常见中文变体。对于非规范命名，我们计划在 P2 阶段增加人工校正入口，允许用户手动指定边界语义（HardWall/AbsorptiveWall/Opening）。"

**当前支持**:
- ✅ AIA 标准图层命名
- ✅ 常见中文变体（墙/墙体/隔墙等）
- ✅ 颜色/线宽过滤

**P2 改进计划**:
- 增加 UI 校正入口（2 周）
- 允许用户手动指定边界语义
- 机器学习辅助识别（P3 阶段考虑）

---

### Q4: 复杂拓扑（嵌套孔洞/非流形几何）能处理吗？

**标准回答**:
> "当前系统使用 DFS + 夹角最小启发式处理简单场景，效果良好。对于复杂拓扑（嵌套孔洞/非流形几何），我们计划在 P2 阶段集成 Halfedge 结构，这是计算几何的标准数据结构。"

**技术细节**（如被追问）:
- 当前方案：DFS + 夹角最小启发式
- 简单闭合环：✅ 完全支持
- 嵌套孔洞（孔中孔）：⚠️ 待验证
- 非流形几何：❌ 不支持

**P2 改进计划**:
- Halfedge 结构集成（2 周，P0 优先级）
- 支持嵌套孔洞和非流形几何
- 面枚举算法优化（O(n²) → O(n)）

**标准回答**:

**RecoverySuggestion 示例**:
```rust
pub struct RecoverySuggestion {
    pub action: String,
    pub config_change: Option<(String, serde_json::Value)>,

}
```

**错误代码**:
- `E001`: 环未闭合 → 建议端点吸附
- `E002`: 自相交 → 建议交点切分
- `E003`: 孔洞在外边界外 → 建议重新检查几何
- `W001`: 短边 → 警告，建议清理
- `W002`: 尖角 → 警告，建议平滑

## 五、核心卖点总结

### 技术优势

1. **EaaS 架构设计**: 服务化拆分清晰，契约明确，可测试性强
2. **220+ 测试**: 包含 E2E 测试和用户故事测试，不是凑数的
3. **性能基线**: 有基线文件、有 CI 回归检测
4. **错误恢复建议**: RecoverySuggestion 带 config_change 和 priority
5. **配置预设模板**: 5 个场景预设 + --profile 命令行支持

### 功能优势

1. **DXF 解析完整**: 嵌套块、NURBS、智能图层
2. **PDF 矢量化**: 自动矢量化（线条清晰图纸）
3. **拓扑构建**: R*-tree 空间索引，O(n log n)
4. **交互 API**: 后端完整实现（选边追踪、圈选区域等）

## 六、限制说明总结

### 需要坦诚说明的限制

1. **交互 UI 前端**: 计划于 P2 阶段（3 周）
2. **光栅 PDF**: 仅适用于线条清晰的图纸
3. **复杂拓扑**: 需要 Halfedge 集成（P2 阶段）
4. **语义标注**: 依赖图层命名规范（P2 阶段增加 UI 校正）

- 不要承诺 100 分，但要让甲方相信 P2 阶段能拿到 95 分
