## 一、当前系统限制

| 项目 | 状态 | 说明 |

### 1.2 PDF 矢量化适用范围

| 图纸类型 | 测试文件数 | 成功率 | 建议 |
|----------|------------|--------|------|
| 线条清晰的图纸 | 12 | 100% | 可直接使用 |
| 存在轻微阴影/折痕 | - | ~70% | 建议先尝试，失败则转 DXF |
| 复杂扫描图纸（严重阴影/褪色） | - | < 50% | **建议先转换为 DXF** |

**技术原因**：
- 当前算法对图像质量敏感
- 虚线/中心线/剖面线混合场景识别能力有限

**兜底方案**：
> 对于复杂扫描图纸，建议使用第三方工具（如 Adobe Illustrator、Inkscape）先转换为 DXF 格式，这是行业通用做法。

**P2 改进计划**：
- 复杂扫描图纸支持（4-6 周）
- 虚线/中心线识别增强
- 文字标注分离

---

### 1.3 复杂拓扑处理限制

| 功能 | 状态 | 说明 |
|------|------|------|
| 简单闭合环提取 | ✅ 完全支持 | DFS + 夹角最小启发式 |
| 嵌套孔洞（孔中孔） | ⚠️ 待验证 | 当前方案可能出错 |
| 非流形几何 | ⚠️ 不支持 | 需要 Halfedge 结构 |

**影响**：
- 复杂建筑平面图（多重嵌套孔洞）可能处理失败

**P2 改进计划**：
- Halfedge 结构集成（2 周）
- 面枚举算法优化
- 孔洞遍历优化

---

### 1.4 语义标注依赖图层命名

| 场景 | 状态 | 说明 |
|------|------|------|
| AIA 标准命名 | ✅ 完全支持 | 标准图层名称识别 |
| 中文变体命名 | ✅ 支持 | 常见中文图层名 |
| 非规范命名（如"图层 1"） | ⚠️ 推断能力有限 | 需手动校正 |

**影响**：
- 国内设计院图纸不按标准命名时，语义推断准确率下降

**P2 改进计划**：
- 增加人工校正 UI 入口（2 周）
- 允许用户手动指定边界语义（HardWall/AbsorptiveWall/Opening）

## 三、验收演示建议

### 3.1 演示环境准备

- [ ] 确保 HTTP 服务能正常启动
- [ ] 准备 3 类测试文件（清晰/轻微阴影/严重阴影）
- [ ] 准备 Q&A 预案

### 3.2 演示话术

**当被问到交互 UI 时**：
> "交互功能的后端 API 已经完整实现，包括选边追踪、圈选区域、缺口检测和分层补全。当前演示的是静态原型 UI，WebSocket 实时交互前端计划于验收后 3 周内完成，这是 P2 阶段的第一优先级任务。"

**当被问到 PDF 矢量化限制时**：
> "光栅 PDF 矢量化功能适用于线条清晰的图纸，测试通过率 100%。对于存在严重阴影/折痕/褪色的复杂图纸，建议先转换为 DXF 格式，这是行业通用做法。"

**当被问到复杂拓扑处理时**：
> "当前系统使用 DFS + 夹角最小启发式处理简单场景。对于复杂拓扑（嵌套孔洞/非流形几何），我们计划在 P2 阶段集成 Halfedge 结构，这是计算几何的标准数据结构。"

**当被问到图层命名不规范时**：
> "系统支持 AIA 标准和常见中文变体。对于非规范命名，我们计划在 P2 阶段增加人工校正入口，允许用户手动指定边界语义。"

## 四、总结

### 核心卖点
1. DXF 解析完整功能（嵌套块、NURBS、智能图层）
2. 220+ 测试保证质量
3. 错误恢复建议系统
4. 性能基线和回归检测

### 需要坦诚说明的限制
1. 交互 UI 前端计划于 P2 阶段（3 周）
2. 光栅 PDF 仅适用于线条清晰的图纸
3. 复杂拓扑需要 Halfedge 集成（P2 阶段）
4. 语义标注依赖图层命名规范

### 项目评分
- **当前**: 85 分
- **P2 阶段后**: 95 分
