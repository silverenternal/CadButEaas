# 性能基线和回归测试

本目录包含性能基准测试的基线数据和回归检测脚本。

## 使用方法

### 1. 建立性能基线

```bash
# 运行基准测试并保存基线
cargo bench --workspace -- --save-baseline main
```

### 2. 运行回归测试

```bash
# 与基线比较，检测性能回归
cargo bench --workspace -- --baseline main
```

### 3. CI 集成

在 CI 中运行性能回归测试：

```bash
# 如果性能下降超过 10%，返回非零退出码
cargo bench --workspace -- --baseline main --baseline-threshold 10
```

## 回归检测规则

### 1. 时间回归

- **警告阈值**: 性能下降 10-20%
- **失败阈值**: 性能下降 > 20%

### 2. 内存回归

- **警告阈值**: 内存增长 15-30%
- **失败阈值**: 内存增长 > 30%

### 3. 复杂度验证

对于基准测试中的规模序列（100→200→400→800 线段），验证时间增长率是否符合 O(n log n) 预期：

- 100→200: 时间比率应为 1.8-2.2x
- 200→400: 时间比率应为 1.8-2.2x
- 400→800: 时间比率应为 1.8-2.2x

## 基线更新流程

当以下情况发生时，需要更新性能基线：

1. **算法优化**: 性能提升后，更新基线反映新性能水平
2. **功能变更**: 新增功能导致性能变化
3. **版本发布**: 每个主要版本（v0.1.0, v0.2.0）更新基线

更新流程：

```bash
# 1. 在 release 模式下运行基准测试
cargo bench --workspace --release -- --save-baseline v0.1.0

# 2. 将基线数据提交到 git
git add benches/baseline/
git commit -m "chore: 更新性能基线至 v0.1.0"
```

## 性能劣化排查流程

当检测到性能回归时：

1. **定位劣化测试**: 识别哪个基准测试性能下降
2. **查看 git 历史**: 找到最近的代码变更
3. **分析原因**: 算法变更？数据结构变更？
4. **修复或接受**: 
   - 如果是意外劣化，回滚或优化
   - 如果是合理代价（如提升精度），更新基线并说明原因
